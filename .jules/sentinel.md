## 2025-02-18 - [Hardcoded Infura API Key in Frontend Configuration]
**Vulnerability:** A hardcoded API key (`ddd78bc17de648b2a89acf424fbfa8ed`) was present in `src/services/tradingAPIService.ts` and `src/contexts/Web3Context.tsx` as a fallback when `VITE_INFURA_API_KEY` was missing. A dummy fallback string (`your_infura_api_key`) was also present in `src/services/hive_mind/vetalShabarRaksha.js`. This creates a risk of leaking API credentials in the frontend bundle and potentially leading to unauthorized API usage.
**Learning:** Hardcoding credentials as OR (`||`) fallbacks in frontend code means that they will be bundled and exposed to the end-users. Even if it's meant as a "dev fallback", it becomes accessible in production bundles unless explicitly removed.
**Prevention:** Never use hardcoded keys or dummy credential strings as fallbacks in `||` expressions for sensitive values. Instead, check for the existence of the environment variable, and gracefully degrade functionality, use generic public nodes without API keys, or log a warning if the key is missing.
